# TZTI 体制内人格测试 — 设计文档

## 概述

TZTI（Tǐzhì nèi Type Indicator）是一个展示中国体制内生态图景的趣味人格测试网页应用。灵感来自 SBTI 测试，采用"公文皮、互联网魂"的混搭调性——表面正经的官方公文风包裹体制内互联网黑话，形成反差喜感。

## 核心机制

### 4个维度

| 维度 | 左极 | 右极 | 衡量内容 |
|------|------|------|----------|
| 第一维 | **卷 (J)** 内卷拼搏 | **佛 (F)** 佛系摸鱼 | 工作投入度 |
| 第二维 | **圆 (Y)** 圆滑世故 | **刚 (G)** 方正耿直 | 人际处事风格 |
| 第三维 | **显 (X)** 高调显眼 | **隐 (H)** 低调隐身 | 存在感偏好 |
| 第四维 | **稳 (W)** 求稳守成 | **浪 (L)** 敢冲敢闯 | 风险偏好 |

组合产生 2⁴ = 16种体制内人格类型。

### 题目设计

- 共24道题：20道基础题 + 4道加权决胜题
- 每题1个体制内场景 + 4个选项
- 基础题每个选项影响1-2个维度，分值±1~2
- 加权题每个维度一道，分值权重×2，处理边界情况
- UI上不区分基础题和加权题，用户无感知
- 预计答题时间6分钟
- 题目场景需融入中央八项规定相关内容，增加真实感和接地气程度，例如：
  - 公务接待标准、聚餐AA制等场景
  - 节日收礼/送礼的处理方式
  - 落实八项规定自查报告等材料任务
  - 公车使用、办公用房标准等日常合规场景

### 八项规定元素融合

八项规定作为体制内生活的重要背景，在以下层面融入测试：

1. **题目场景**：部分题目直接使用八项规定相关的日常场景作为测试情境
2. **类型描述**：在各类型的详细描述中自然引用，如"局长预备役"熟知各项标准红线，"摸鱼之王"把八项规定当作拒绝应酬的挡箭牌
3. **生存建议**：结合八项规定的现实约束给出建议，增加反差梗和现实感

融合原则：自然嵌入，不生硬说教，保持娱乐调性。

### 发展规划元素融合

体制内高频的政策词汇和发展规划内容作为题目场景的第三类素材：

1. **题目场景**：使用政策学习和落实的日常情境，例如：
   - 领导安排写"新质生产力"调研报告
   - 单位对标"十四五规划"指标做年终总结
   - 开会讨论"高质量发展"落实方案
   - 上级要求学习最新政策文件并写心得体会
2. **选项语言**：选项中自然融入体制内黑话（"对标对表""靶向发力""赋能增效"等），制造答题过程中的会心一笑
3. **类型描述**：部分类型描述引用政策语境，如"局长预备役"的汇报材料里"新质生产力"用得炉火纯青

### 题目场景三大类别

题目场景覆盖体制内日常的三个核心面向，确保测试的立体感和真实感：

| 类别 | 占比 | 示例场景 |
|------|------|----------|
| 人际关系与职场生存 | ~50% | 领导消息回复、办公室政治、同事关系、晋升竞争 |
| 八项规定与日常合规 | ~25% | 公务接待、节日送礼、公车使用、自查报告 |
| 政策学习与发展规划 | ~25% | 写调研报告、学习心得、对标指标、会议发言 |

### 计分机制

- 每个维度用数值轴表示：0为中点，正值偏左极，负值偏右极
- 最终结果：4个维度各取正负方向，组成4字母类型码
- 维度百分比 = 将原始分映射到0%-100%（50%为中点），用于结果页展示

## 页面流程

### 1. 首页

- 顶部：印章风格圆形徽章，内书"体制内人格鉴定"
- 主标题：**T Z T I**（大字，衬线体）
- 副标题：体制内人格测试 / Tǐzhì nèi Type Indicator
- 4个维度标签横排：`卷vs佛` `圆vs刚` `显vs隐` `稳vs浪`
- 说明："24道题，测出你在体制内的真实人格"
- 按钮："开始鉴定"
- 底部小字："* 纯属娱乐，请勿对号入座（虽然你一定会）"

### 2. 答题页

- 顶部进度条 + "第 X/24 题"
- 题目卡片：场景描述 + 4个选项
- 点击选项后短暂高亮反馈（0.3s），自动滑入下一题
- 支持点击进度条回退修改答案

### 3. 结果页

- 标题："组织鉴定书"
- 维度码大字展示（如 JYXW）
- 类型代号 + emoji
- 一句话标签
- 4个维度对比条（左右两极，标注百分比倾向）
- 3-4句详细描述
- 生存建议
- 底部操作栏："重新鉴定" + "生成分享图"

## 视觉设计

### 配色

- 主背景：#1a1a2e（深墨蓝）
- 卡片背景：rgba(255,255,255,0.05)（半透明深色）
- 主色调：#c43a31（朱红）— 强调、按钮、印章
- 点缀色：#d4a847（金色）— 标题、维度标签、进度条
- 正文：#f0e6d3（暖白）

### 字体

- 标题/类型名：Noto Serif SC（衬线体）
- 正文/选项：Noto Sans SC（无衬线）

### 特色元素

- 印章：圆形红色边框 + 篆体风格文字，首页和结果页均出现
- 水墨纹理：背景微弱水墨晕染效果（CSS渐变模拟，不用图片）
- 卡片边框：细金线，hover时微光效果
- 维度条：朱红到金色渐变

### 动效

- 页面切换：淡入淡出（0.4s）
- 答题卡片：左滑出 + 右滑入（0.3s）
- 结果页维度条：从0动画展开到实际值（0.6s，带缓动）
- 按钮：hover微上浮 + 阴影加深

## 分享机制

### 分享图生成

- 用原生 Canvas API 将结果卡片渲染为图片（零依赖）
- 渲染内容：类型码 + 代号 + 标签 + 4个维度条 + 底部水印

### URL参数编码

- 格式：`?r=JYXW-73-82-45-91`
- JYXW = 类型码，后面4个数字 = 4个维度百分比
- 打开带参数链接 → 直接展示结果页 + 顶部引导"测测你是什么类型"

## 技术架构

### 技术方案

纯静态单页应用（HTML + CSS + JS），零依赖，零构建步骤。

### 文件结构

```
tzti/
├── index.html      # 主页面，3个section切换
├── style.css       # 所有样式
├── data.js         # 题目数据 + 16种类型定义
└── app.js          # 状态管理、答题逻辑、结果计算、分享功能
```

### 兼容性

- 移动端优先，微信内置浏览器兼容
- CSS 用 flexbox 布局（不依赖 grid）
- 不使用 ES module，直接 script 标签引入
